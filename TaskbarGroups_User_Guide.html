第一章：軟體概述與安裝指南

1.1 軟體簡介與設計初衷

在日常的 Windows 系統使用中，我們經常會需要開啟許多不同的應用程式、開發工具或腳本。如果將它們全部釘選在 Windows 工作列上，將會佔用寶貴的螢幕空間，使工作列變得擁擠不堪；如果放在桌面上，又會被開啟的視窗遮擋，降低使用效率。

自訂工作列群組管理器 (TaskbarGroups) 是一個專為解決此痛點而設計的輕量級開源工具。它能讓您將多個相關的應用程式歸納至同一個「群組」中。例如，您可以建立一個名為「開發工具」的群組，內含 Visual Studio、VS Code、Git Bash 和 SQL Server Management Studio；再建立一個「辦公軟體」群組，內含 Word、Excel 與 PowerPoint。

本工具將這些群組封裝為單個 Windows 捷徑。將其釘選到工作列後，只需點擊一下，便會於工作列上方（或依據工作列的位置）彈出一個美觀的暗色半透明圓角浮動選單，點選其中的應用程式即可快速啟動。它不僅極大化地節省了螢幕空間，更為您的視窗桌面整理帶來了極高的便利性與美感。

核心設計理念
  • 無依賴性： 完全採用 Windows 原生 API 與 .NET Framework 核心庫，無需安裝額外的執行環境或第三方 DLL。
  • 極簡輕量： 編譯後主程式小於 30 KB，運行期間幾乎不佔用任何系統 CPU 或記憶體資源。
  • 原生美感： 提供精心設計的 Windows 暗色風格懸停與過渡動畫，適配 Windows 的現代 UI 美學。

1.2 系統需求與編譯部署

本工具基於 C# 與 .NET Framework 4.0 編寫，因此與絕大多數 Windows 系統相容。

系統環境要求

  • 作業系統：Windows 10 / Windows 11 (64位元版本最佳)
  • 軟體框架：.NET Framework 4.0 或更高版本 (Windows 10/11 系統已預裝)
  • 系統權限：建議使用標準使用者或系統管理員權限運行

原始碼編譯指南

工作區中已貼心地為您配置好了編譯批次檔 compile.bat。您無需安裝龐大的 Microsoft Visual Studio，即可直接利用系統內建的 .NET 編譯器將原始碼編譯為可執行檔。

1 檢查環境與編譯路徑
預設情況下，編譯指令將會調用系統路徑：
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe。請確保此編譯器存在。
2 執行編譯指令
雙擊點擊工作區根目錄下的 compile.bat，或者在終端機中執行該批次檔： 
d:\antigravity\TaskbarGroups> compile.bat
3 驗證編譯結果
若編譯成功，終端機將顯示 [SUCCESS] Compilation succeeded! Created TaskbarGroups.exe.，並在根目錄下生成 TaskbarGroups.exe 執行檔。
注意：
若執行 compile.bat 時提示 csc.exe not found 錯誤，代表您的系統 .NET Framework 安裝路徑可能與預設路徑不同，或者您正在使用 32 位元的 Windows 系統。此時請編輯 compile.bat 將其編譯路徑修改為 Framework（而非 Framework64）。

1.3 目錄與檔案結構說明

本軟體執行時，其相關的所有設定、快取圖示以及捷徑均儲存在使用者的 AppData 目錄下。這樣做的好處是保證了軟體本體的「綠色免安裝性」，且多個系統使用者之間的設定完全獨立。

主程式在執行過程中會自動在以下路徑建立檔案夾結構： 

%USERPROFILE%\AppData\Roaming\CustomTaskbarGroups\

詳細的檔案目錄架構說明如下表所示：

目錄/檔案路徑 檔案類型與格式 功能與用途說明
groups\ 設定檔 (.cfg) 存放各個應用程式群組的設定資料。檔案名稱即為群組名稱。
shortcuts\ 捷徑 (.lnk) 存放管理器生成的 Windows 捷徑。這些捷徑是用於釘選到工作列的媒介。
icons\ 圖示 (.ico / .png) 存放從應用程式中自動提取出的圖示 (PNG) 以及群組的專屬圖示 (ICO)。
hotkeys.cfg 設定檔 (.cfg) 存放全域快捷鍵的配置資訊（格式為：群組名稱|fsModifiers|vkCode|鍵盤文字）。
目錄速查提示
您可以在「自訂工作列群組管理器」的介面底部點擊 「打開捷徑資料夾」 按鈕，系統將自動開啟 shortcuts\ 資料夾，方便您快速存取產生的捷徑。

第二章：管理器模式操作手冊 (Manager Mode)

2.1 啟動管理器介面

當您直接執行 TaskbarGroups.exe（不帶任何命令行參數啟動）時，軟體會自動偵測並進入 「管理器模式」。

管理器介面分為左右兩個大區塊：

  • 左側「群組清單」： 顯示所有已建立的群組。您可以在此進行新增群組、選擇群組，或點選刪除群組。
  • 右側「編輯詳細設定」： 當您點選左側的群組後，右側會自動載入該群組的名稱、圖示以及所包含的應用程式列表，並允許您在此進行細部調整。

2.2 新增與刪除應用程式群組

新增群組

  1. 點擊左下角的 「新增群組」 按鈕。
  2. 在彈出的對話框中輸入新群組的名稱（例如：設計軟體）。名稱中不能包含 Windows 檔案命名禁用字元（如 \ / : * ? " < > |）。
  3. 點擊「確定」後，系統會自動在 groups\ 目錄下建立對應的設定檔，並在左側清單中載入該群組以供編輯。

刪除群組

  1. 在左側「群組清單」中選取您要刪除的群組.
  2. 點擊左下角的 「刪除群組」 按鈕。
  3. 系統會顯示確認對話框。確認後，將會一併刪除該群組的 .cfg 設定檔以及 shortcuts\ 中的 Windows 捷徑。

2.3 新增應用程式與參數設定

選取一個群組後，您可以在右側的應用程式列表欄位中點擊 「新增應用程式」 按鈕。

1 選擇執行檔
點擊按鈕後會彈出檔案選擇對話框。請瀏覽您的硬碟並選取目標應用程式的可執行檔（如 .exe、.bat、.cmd 檔案）。
2 自動提取圖示
選取檔案後，軟體底層會自動調用 Windows Shell API 提取該 EXE 的關聯圖示，並將其自動轉換為 PNG 格式，儲存於 icons\ 目錄下。
3 編輯參數 (選填)
如果該程式在啟動時需要附加參數（例如 Chrome 需要附加 --incognito 無痕參數，或者腳本需要特定引數），您可以直接雙擊清單中的「啟動參數」欄位進行鍵入與編輯。

2.4 設定群組圖示與排序調整

設定與修改群組圖示

群組圖示是顯示在 Windows 工作列上的圖標。一個美觀、具有識別性的圖示能大幅提升工作效率。

  • 預設圖示： 如果您沒有為群組設定圖示，系統會自動生成一個預設的「現代化 2x2 藍色網格圖標」。
  • 自訂圖示： 點擊右側上方的 「選擇圖示」 按鈕，您可以載入自己的 .ico、.png、.jpg 等格式影像。
技術貼心設計
Windows 捷徑系統限制必須使用 .ico 格式作為捷徑圖示。如果使用者選擇了 PNG 或 JPG 等非 ICO 格式，管理器在儲存時會自動於背景將其轉換為標準的 ICO 圖示，確保能被 Windows 系統正常識別與顯示。

調整應用程式順序

彈出浮動選單時，應用程式會依據清單中的上下順序排列。您可以使用清單右側的 「上移」 與 「下移」 按鈕，即時調整各個應用程式在彈出選單中的前後位置。

2.5 儲存與產生捷徑

編輯完成後，請點擊右下角醒目的藍色按鈕：「儲存並產生工作列捷徑」。軟體將執行以下背景程序：

  1. 將當前設定值寫入到 %APPDATA%\CustomTaskbarGroups\groups\[群組名].cfg 設定檔中。
  2. 如果修改了群組名稱，軟體會自動清理舊的設定檔與捷徑，避免檔案殘留。
  3. 在 shortcuts\ 資料夾下生成一個同名的 Windows 捷徑 (.lnk)。此捷徑的目標指向 TaskbarGroups.exe，並將群組名稱作為參數帶入（如：TargetPath: "...TaskbarGroups.exe" Arguments: "開發工具"）。
  4. 儲存完畢後，軟體會自動彈出成功提示，並主動為您開啟 shortcuts 捷徑資料夾。

2.6 全域快捷鍵設定與捕獲

如果您想用鍵盤快速召喚選單，可以在右側面板設定全域快捷鍵：

  1. 選取群組後，點擊右側的 「全域快捷鍵」 輸入框。
  2. 在鍵盤上按下您的快捷鍵組合（例如 Ctrl + Alt + K），輸入框會自動捕捉並顯示。
  3. 若要清除該快捷鍵，請在該輸入框中按下 Backspace 或 Delete 鍵，欄位會恢復為「無」。
  4. 點擊「儲存並產生工作列捷徑」後，該快捷鍵就會即時向 Windows 系統註冊，隨時待命。

第三章：啟動器與工作列整合 (Launcher Mode)

3.1 將群組捷徑釘選至工作列

由於 Windows 10/11 的保護限制，普通的捷徑無法直接透過拖曳釘選至工作列，請嚴格按照以下標準步驟進行操作：

1 開啟捷徑資料夾
在管理器中點擊「儲存」或點選「打開捷徑資料夾」，系統將會開啟：
%APPDATA%\CustomTaskbarGroups\shortcuts。
2 使用右鍵選單釘選
對著您剛才建立的群組捷徑（例如 開發工具.lnk）點擊滑鼠右鍵。
  • Windows 10： 直接點選選單中的 「釘選到工作列」。
  • Windows 11： 點選 「顯示更多選項」 → 點選 「釘選到工作列」。
3 拖曳至理想位置
此時，您自訂的群組圖示便會出現在 Windows 工作列上。您可以按住滑鼠左鍵，將其拖曳移動到您最順手的位置。

3.2 彈出式浮動選單操作

當您點擊工作列上的群組圖示時，軟體會以參數啟動並進入 「啟動器模式」。此時螢幕上會顯示一個專屬設計的浮動選單：

  • 半透明暗黑設計： 選單採用現代 Windows 風格的深色調（RGB: 30, 30, 30），邊角處進行了 12 像素的圓角化（Round Corner）裁切，並附帶精細的硬體陰影效果（Drop Shadow）。
  • 格線配置： 應用程式以 5 行為最大上限排版。若有 6 個應用程式，將自動排列為 2 行（5 + 1），版面維持整齊對稱。
  • 極速響應： 點擊選單中的任何應用程式，會立刻在背景啟動該程式，並同時關閉浮動選單。
  • 無打擾退出： 若您不小心點開了選單但又不想啟動任何程式，只需在選單以外的任何螢幕區域點選滑鼠，或是按下鍵盤的 Escape 鍵，選單便會自動優雅關閉，絕不干擾您的手頭工作。
貼心操作提示
本軟體在彈出浮動選單時，不會在 Windows 工作列上顯示 any 視窗圖示，這保證了工作列的視覺乾淨度，讓群組選單看起來完全像是 Windows 原生的系統選單。

3.3 智慧位置定位算法剖析

每位使用者擺放 Windows 工作列的位置與螢幕解析度皆不相同。為了確保浮動選單能完美貼合工作列，且絕對不會超出螢幕邊界，軟體底層設計了一套智慧定位算法。

程式啟動時會調用 Screen.FromPoint(Cursor.Position) 取得當前滑鼠所在的螢幕，並獲得該螢幕的「總邊界 (Bounds)」與「工作區 (Working Area，即扣除工作列後的範圍)」。

定位邏輯判斷：

  1. 工作列位於螢幕下方：
    判斷條件：WorkingArea.Bottom < Bounds.Bottom
    定位計算：選單的底邊將自動貼齊工作區底端，並向上偏移 8 像素：y = WorkingArea.Bottom - Form.Height - 8。
  2. 工作列位於螢幕上方：
    判斷條件：WorkingArea.Top > Bounds.Top
    定位計算：選單的頂邊貼齊工作區頂端，並向下偏移 8 像素：y = WorkingArea.Top + 8。
  3. 工作列位於螢幕左方：
    判斷條件：WorkingArea.Left > Bounds.Left
    定位計算：選單的左邊貼齊工作區左端，並向右偏移 8 像素：x = WorkingArea.Left + 8。
  4. 工作列位於螢幕右方：
    判斷條件：WorkingArea.Right < Bounds.Right
    定位計算：選單的右邊貼齊工作區右端，並向左偏移 8 像素：x = WorkingArea.Right - Form.Width - 8。

計算完成後，系統會執行一道保護檢測，強制限制 x 與 y 坐標必須落在 WorkingArea 之內，從而徹底避免了在多螢幕或特殊解析度下選單被切掉的問題。

3.4 側邊分頁快速切換

當您的系統中存在 2 個或以上的群組時，彈出的浮動選單左側會自動載入一個動態分頁側邊欄：

  • 群組無縫切換： 側邊欄會以高質感的圓角主題色字首圖示列出所有已建立的群組。您只需將滑鼠移至側邊欄並點選，即可動態將該選單切換至該群組。
  • 視窗大小自適應： 點擊切換群組時，選單視窗的高度、寬度以及 GDI 圓角範圍會依照新群組的內容即時動態重新計算，配合淡入動畫，切換流暢、完全不拖泥帶水。

3.5 系統匣常駐與背景監聽

為了實現全域快捷鍵的「毫秒級」無感響應，管理器提供了系統匣（System Tray / NotifyIcon）常駐執行模式：

  • 背景執行： 當點選群組管理器右上角的 X 關閉時，視窗會自動隱藏並退入系統右下角系統匣常駐執行。此時，全域快捷鍵召喚機制會保持高度活躍。
  • 系統匣選單： 滑鼠右鍵點擊系統匣小圖示，會彈出「開啟管理器」與「徹底退出」選項。選擇「徹底退出」時，程式會釋放所有全域快捷鍵並徹底關閉進程。

第四章：架構設計與底層原理剖析

4.1 雙模式執行機制

本軟體精妙地將「管理器 (Manager)」與「啟動器 (Launcher)」整合於單一的 C# 檔案與 TaskbarGroups.exe 中。在 Main 進入點中，透過判斷命令行參數 args 的長度來切換運作模式： 

[STAThread]
static void Main(string[] args)
{
    Application.EnableVisualStyles();
    Application.SetCompatibleTextRenderingDefault(false);

    if (args.Length > 0)
    {
        // 啟動器模式：傳入了群組名稱參數
        string groupName = args[0];
        LauncherForm launcher = new LauncherForm(groupName);
        if (!launcher.LoadFailed)
        {
            Application.Run(launcher);
        }
    }
    else
    {
        // 管理器模式：直接雙擊執行
        Application.Run(new ManagerForm());
    }
}

4.2 設定檔儲存格式 (CFG)

設定檔儲存在 groups\ 資料夾下，副檔名為 .cfg，編碼為 UTF-8。為了追求效能，軟體沒有引入大型的 XML 或 JSON 函式庫，而是採用了非常高效率的自訂文字行格式。

設定檔的結構規則如下：

  • 第一行： 群組名稱。
  • 第二行： 群組專屬圖示的絕對路徑。
  • 第三行及以後： 該群組所含的應用程式資訊。每一行代表一個應用程式，欄位之間以 | 符號進行分割。

欄位結構：

[程式顯示名稱]|[執行檔絕對路徑]|[啟動引數(選填)]|[圖示檔案路徑(選填)]

設定檔範例 (開發工具.cfg)：

開發工具
C:\Users\USER\AppData\Roaming\CustomTaskbarGroups\icons\開發工具_group.ico

4.3 免依賴捷徑反射生成法

在 Windows 中建立捷徑 (.lnk) 通常需要引用 COM library (Windows Script Host Object Model)。為了避免在編譯時需要綁定額外的 Interop DLL 或造成不同作業系統版本的相容性問題，本軟體使用了「動態反射 (Dynamic Reflection)」技術來調用 Windows 的 COM 元件。

以下是底層的實現原始碼，展示了軟體如何優雅地在零相依性的情況下動態調用 WScript.Shell： 

private void CreateShortcut(string groupName, string iconPath)
{
    string shortcutPath = Path.Combine(GroupConfig.GetShortcutsDir(), groupName + ".lnk");
    string targetExe = Application.ExecutablePath;

    try
    {
        // 透過 ProgID 動態取得 WScript.Shell 類型並建立執行個體
        Type shellType = Type.GetTypeFromProgID("WScript.Shell");
        object shell = Activator.CreateInstance(shellType);
        
        // 反射調用 CreateShortcut 方法
        object shortcut = shellType.InvokeMember("CreateShortcut", 
            System.Reflection.BindingFlags.InvokeMethod, null, shell, new object[] { shortcutPath });
        
        // 動態反射設定捷徑的各項屬性
        Type shortcutType = shortcut.GetType();
        shortcutType.InvokeMember("TargetPath", System.Reflection.BindingFlags.SetProperty, null, shortcut, new object[] { targetExe });
        shortcutType.InvokeMember("Arguments", System.Reflection.BindingFlags.SetProperty, null, shortcut, new object[] { "\"" + groupName + "\"" });
        shortcutType.InvokeMember("WorkingDirectory", System.Reflection.BindingFlags.SetProperty, null, shortcut, new object[] { Path.GetDirectoryName(targetExe) });
        
        if (!string.IsNullOrEmpty(iconPath) && File.Exists(iconPath))
        {
            shortcutType.InvokeMember("IconLocation", System.Reflection.BindingFlags.SetProperty, null, shortcut, new object[] { iconPath });
        }
        
        // 儲存捷徑檔案
        shortcutType.InvokeMember("Save", System.Reflection.BindingFlags.InvokeMethod, null, shortcut, null);
    }
    catch (Exception ex)
    {
        throw new Exception("無法建立 Windows 捷徑：" + ex.Message);
    }
}

第五章：常見問題與排錯指南 (FAQ)

Q1：為什麼點擊捷徑後，彈出的視窗一閃而過，或者提示找不到群組設定？

答： 這通常是由於捷徑中的參數被損壞或遺失所致。請確認捷徑的「目標」屬性最後是否帶有以引號包圍的群組名稱作為參數（例如 "開發工具"）。若丟失，最簡單的解決方法是打開管理器 (TaskbarGroups.exe)，選取該群組並重新點擊「儲存並產生工作列捷徑」按鈕來重新生成捷徑。

Q2：我可以在群組中加入批次檔 (.bat) 或命令提示字元 (.cmd) 嗎？

答： 可以。在新增應用程式時，將檔案選擇器過濾器切換為「所有檔案 (*.*)」即可選擇 .bat、.cmd 或任何其他可執行檔案。但需要特別注意的是，此類腳本檔案可能沒有內嵌圖示，軟體會無法為其自動提取 PNG 圖示，此時系統會自動使用預設的空圖示。

Q3：為什麼我把群組的捷徑拖曳到工作列上，卻無法釘選？

答： Windows 系統基於安全性考量，不允許使用者直接將自訂的 .lnk 捷徑拖曳到工作列上進行釘選。請一定要對著捷徑點擊滑鼠右鍵，在右鍵選單中點選「釘選到工作列」。

Q4：當我刪除管理器中的群組時，我加入裡面的應用程式會被刪除嗎？

答： 不會。刪除群組只會刪除您的群組設定檔 (.cfg) 以及捷徑 (.lnk)，這只是刪除了「啟動清單關係」，絕不會對您安裝在電腦中的任何原始應用程式檔案造成影響，請放心使用。

Q5：如何備份或轉移我的群組設定？

答： 您只需要將 %APPDATA%\CustomTaskbarGroups\ 資料夾複製備份即可。移置新電腦時，將此資料夾貼回相同的 AppData 路徑下，並重新編譯執行本程式，即可完美回復您所有的自訂工作列群組。
